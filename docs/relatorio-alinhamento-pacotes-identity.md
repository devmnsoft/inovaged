# Relatório de alinhamento de pacotes Identity e ASP.NET Core

| Pacote | Versão anterior | Versão nova | Motivo | Consumidores | Risco | Teste realizado |
|---|---:|---:|---|---|---|---|
| `Microsoft.AspNetCore.Cryptography.KeyDerivation` | `8.0.11` em ambiente/local/cache ou referência obsoleta anterior; `8.0.27` no arquivo atual | `8.0.27` | Alinhar com `Microsoft.Extensions.Identity.Core 8.0.27`, que exige `KeyDerivation >= 8.0.27`; o projeto chama `KeyDerivation.Pbkdf2` diretamente. | `InovaGed.Infrastructure/Security/Pbkdf2PasswordHasher.cs` | Baixo: atualização patch do .NET 8. | `rg` confirmou que não há referência ativa `8.0.11` fora de documentação histórica; `dotnet` não está instalado no ambiente para restore local. |
| `Microsoft.Extensions.Identity.Core` | `8.0.27` | `8.0.27` | Mantido na mesma linha patch do KeyDerivation para evitar NU1605. | `InovaGed.Infrastructure`, transitivamente `InovaGed.Web`, `WebGed.WebApi`, testes | Baixo. | Inspeção de `.csproj` e validação por workflow adicionada. |
| `Microsoft.AspNet.Identity.Core` | `2.2.4` | Removido | Pacote legado ASP.NET Identity (System.Web) não tinha usos `Microsoft.AspNet.Identity` no código e não é coerente com `net8.0`. | Nenhum uso direto localizado. | Médio se algum código externo dependia transitivamente; mitigado por build CI. | `rg -n "Microsoft\.AspNet\.Identity" --glob '*.cs'` não retornou consumidores. |
| `Microsoft.AspNetCore.Authorization` | `2.3.0` | `FrameworkReference Microsoft.AspNetCore.App` | Evita pacote ASP.NET Core 2.x em projetos `net8.0`; Authorization faz parte do shared framework ASP.NET Core 8. | Controllers, handlers e serviços com `Microsoft.AspNetCore.Authorization`. | Baixo/médio: troca de referência de pacote legado por shared framework suportado. | Workflow executa restore/build/test. |
| `Microsoft.AspNetCore` | `2.3.0` | `FrameworkReference Microsoft.AspNetCore.App` | Remove pacote meta legado 2.x do projeto Application e usa shared framework compatível com .NET 8. | Interfaces e modelos de autorização na camada Application. | Médio por ampliar dependência de shared framework na biblioteca. | Workflow executa restore/build/test. |
| `Microsoft.AspNetCore.Mvc.Core` | `2.3.0` | Removido | Projeto Web usa `Microsoft.NET.Sdk.Web`, que referencia `Microsoft.AspNetCore.App`; manter MVC 2.x mistura stacks. | Controllers/views do `InovaGed.Web`. | Baixo. | Workflow executa restore/build/test. |
| `Microsoft.AspNetCore.Authentication.Certificate` | `8.0.27` | `8.0.27` | Mantido na linha patch .NET 8 já utilizada. | Autenticação por certificado no Web. | Baixo. | Workflow executa restore/build/test. |
| `Microsoft.AspNetCore.Authentication.JwtBearer` | `8.0.27` | `8.0.27` | Mantido na linha patch .NET 8 já utilizada. | `WebGed.WebApi`. | Baixo. | Workflow executa restore/build/test. |
